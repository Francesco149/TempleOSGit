                               Compiler Options

Use [C:/Compiler/CMisc.CPP.Z,1] Option().  You might need to do #exe
{Option();}.

[C:/Kernel/KernelA.HPP.Z,1512] OPTf_GLBLS_ON_DATA_HEAP without this option,
global vars are placed in the 
code heap which is limited to 2 Gig.  In AOT modules, global vars take-up
room 
in the .BIN file, so you might want to use this option, instead.  You might 
wish to turn it on and off around specific vars.  A disadvantage of data heap 
global vars in AOT modules is they can't be initialized.

[C:/Kernel/KernelA.HPP.Z,1509] OPTf_EXTERNS_TO_IMPORTS and
[C:/Kernel/KernelA.HPP.Z,1510] OPTf_KEEP_PRIVATE are strange options, you'll 
never need.  They're to allow the same header file for Kernel to act as extern
s when compiling itself and imports when compiled by AOT modules.

[C:/Kernel/KernelA.HPP.Z,1506] OPTf_WARN_UNUSED_VAR	warning if unused var. 
It is applied to functions.

[C:/Kernel/KernelA.HPP.Z,1507] OPTf_WARN_PAREN		warning if parenthesis are
not needed.

[C:/Kernel/KernelA.HPP.Z,1508] OPTf_WARN_DUP_TYPES	warning if dup local var
type stmts.

[C:/Kernel/KernelA.HPP.Z,1511] OPTf_NO_REG_VAR forces all function local vars
to the stk not regs.  Applied 
to functions.

[C:/Kernel/KernelA.HPP.Z,1514] OPTf_NO_BUILTIN_CONST Disable 10-byte float
consts for pi, log2_10, log10_2, 
loge_2.  Applied to functions.
